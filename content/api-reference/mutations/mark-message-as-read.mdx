---
title: Mark message as read
description: Mark a message activity as read
---

This request allows us to update the status of the message activity to "DONE".

## Request

### Query

```graphql
mutation MarkMessageAsRead($input: MarkMessageAsReadInput!) {
  markMessageAsRead(input: $input) {
    activity {
      id
      status
    }
  }
}
```

### Variables

An example `activity_id` is "Activity/ZIRzBxe37Zs9sLnk7LWJY"

```json
{
  "input": {
    "activity_id": "{{MESSAGE_ACTIVITY_ID}}"
  }
}
```

### Request Headers

An additional request header is required, to indicate the user id of the message reader. If a patient is reading the message, for example, then this user id is the patient user's id.

```json
{
  "X-User-ID": "{{USER_ID}}"
}
```
